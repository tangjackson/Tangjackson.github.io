Attention Is All You Need — Notes & Intuitions

January 15, 2024 · 8 min read

The Transformer (Vaswani et al., 2017) is now the backbone of almost everything in NLP and beyond. These are my personal notes after reading the paper carefully — focused on building up intuition rather than just restating formulas.

Why Attention?

Before Transformers, sequence models were mostly RNN-based (LSTM, GRU). The core problem: RNNs process tokens sequentially, so the path between position i and position j has length O(|i−j|). Long-range dependencies are hard to learn because the gradient has to flow through many steps.

Self-attention fixes this: every position attends to every other position in O(1) steps. The trade-off is O(n²) computation in sequence length, which is fine for typical sentence lengths but painful for very long contexts.

"Attention is a soft lookup in a database of key-value pairs, where the query is the current token's representation."

Scaled Dot-Product Attention

Given queries Q, keys K, and values V (all matrices):

Attention(Q, K, V) = softmax( QKᵀ / √dₖ ) · V

The scale factor √dₖ is the key trick. When dₖ is large, dot products grow in magnitude, pushing softmax into regions with tiny gradients. Dividing by √dₖ keeps the variance roughly constant.

Intuition for Q, K, V

  • Query (Q): "What am I looking for?"
  • Key (K): "What do I offer to be found?"
  • Value (V): "What do I actually contribute if selected?"

This is literally a soft retrieval system. The attention weights tell each token how much to "borrow" from every other token's value.

Multi-Head Attention

Rather than one attention function with dmodel-dimensional Q, K, V, multi-head attention runs h parallel attention functions with dₖ = dmodel/h each:

MultiHead(Q, K, V) = Concat(head₁, …, headₕ) · W^O

Different heads can learn different types of relationships simultaneously — syntactic dependencies in one head, semantic similarity in another, coreference in a third. The paper uses h=8 heads.

Positional Encoding

Self-attention has no notion of position — swapping two tokens gives the same output. To fix this, a positional encoding PE(pos, i) is added to each embedding before the first layer: 

PE(pos, 2i)   = sin(pos / 10000^(2i/dmodel))
PE(pos, 2i+1) = cos(pos / 10000^(2i/dmodel))

Why sinusoids? They let the model generalize to sequence lengths not seen during training, because relative positions can be expressed as linear functions of the encoding at any fixed offset.

The Full Architecture

The Transformer is an encoder-decoder stack:

  • Encoder: N=6 identical layers, each with multi-head self-attention + FFN, with residual connections and layer norm.
  • Decoder: N=6 layers with masked self-attention (causal), cross-attention to encoder output, and FFN.
  • The FFN in each layer is a simple 2-layer MLP: FFN(x) = max(0, xW₁ + b₁)W₂ + b₂, with inner dimension 2048.

Key Takeaways

  • Attention solves the long-range dependency problem elegantly.
  • Multi-head lets the model attend to multiple subspaces simultaneously.
  • Residual connections + layer norm are critical for training deep stacks.
  • Positional encodings inject order information without breaking the parallel computation.

My Questions / Open Points

  • Why layer norm instead of batch norm? (Hint: sequence lengths vary; batch norm over the sequence dimension doesn't make sense.)
  • What do different attention heads actually learn? (See: Clark et al., 2019)
  • The O(n²) bottleneck — how do sparse/linear attention variants address it?
